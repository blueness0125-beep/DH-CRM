## 4. 핵심 비즈니스 로직

### 4.1 갱신일 처리 (중요!)
`car_insurance_data.갱신일`은 **MM-DD 형식** (연도 없음). "next occurrence" 계산 필수.

```typescript
// 다음 갱신일 계산 (사용 위치: renewals/route.ts, dashboard/stats/route.ts, car-insurance/route.ts)
function getNextRenewalDate(mmdd: string, from: Date): Date {
  const [mm, dd] = mmdd.split("-").map(Number)
  const thisYear = from.getFullYear()
  const renewalThisYear = new Date(thisYear, mm - 1, dd)
  return renewalThisYear < from ? new Date(thisYear + 1, mm - 1, dd) : renewalThisYear
}
```

### 4.2 대시보드 통계
- **carRenewal45Count**: car_insurance_data 전체 조회 → JS 필터 (SQL 불가, MM-DD 형식)
- **sangryungilThisMonth**: 생일 + 60일 = 상령일, 이번 달에 해당하는 고객 수
- **upcomingBirthdays**: 오늘부터 14일 이내 생일 고객

### 4.3 자동차보험 갱신 목록
- 기본 정렬: `fullRenewalDate` (다음 갱신일 기준 오름차순)
- 필터: upcoming45 (45일 이내) / month:MM (특정 월) / all
- 한 고객이 여러 차량 보유 가능 → 차량정보는 멀티라인 텍스트

### 4.4 이미지 경로 파싱
`비교표경로`, `가입정보경로`, `이미지경로` 컬럼은 개행(`\n`)으로 구분된 URL 목록.
```typescript
parseUrls(raw: string | null): string[] {
  return raw?.split("\n").map(u => u.trim()).filter(Boolean) ?? []
}
```

### 2026-04-17 — 자동차보험 갱신 버그 수정 (FK 및 JOIN 복원)

**문제:** `/admin/renewals/car-insurance` 페이지에서 목록이 비어 있음

**원인:** `car_insurance_data` 테이블이 수동 생성되어 `customers` 테이블과의 FK 제약(Foreign Key Constraint)이 선언되지 않음 → PostgREST가 JOIN 거부

**조치:**
1. 임시 조치: `customers` JOIN 제거, `고객명`/`연락처`/`customer_id` 직접 사용 (커밋 `1ef8da9`)
2. Supabase SQL Editor에서 FK 제약 추가:
   ```sql
   ALTER TABLE car_insurance_data
     ADD CONSTRAINT fk_car_insurance_data_customer
     FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE SET NULL;
   ```
3. FK 확인 후 `customers` JOIN 복원 (커밋 `819e3f9`)

**결과:** JOIN 정상, 고객명 클릭 시 고객 상세 페이지 이동 동작

> **학습:** `car_insurance_data`는 외부 시스템에서 수동 생성된 테이블. 신규 컬럼/관계 추가 시 항상 Supabase SQL Editor에서 수동 실행 필요.

### 자동차보험 관련 작업 시
1. `갱신일`은 항상 MM-DD 형식. 연도 계산 필요 시 `getNextRenewalDate()` 사용

3. `car_insurance_data`에 INSERT하지 않음 (외부 시스템 영역)
